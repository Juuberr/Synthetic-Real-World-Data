## Project Goal (from assignment PDF)

The central question: **when does generated synthetic data preserve what matters, and when does it fail?**

Evaluation must cover three axes:
- **Utility** — train on synthetic, test on real; measure F1, AUC, or similar
- **Fidelity** — compare distributions, correlations, category frequencies, structural properties
- **Privacy / memorization** — duplicate rate, nearest-neighbor similarity, rare-record copying, membership leakage

The final report must include concrete failure cases (invalid rows, lost minority classes, broken constraints) and clearly state which structure is preserved vs. lost.

### Project Phases (from assignment)
- **Phase 0**: Read the DeepMind blogpost for inspiration
- **Phase 1**: Get 2 models + 1 dataset up and running
- **Phase 2**: Collect a battery of datasets and models
- **Phase 3**: Factor analysis — study tradeoffs when targeting one quality (fidelity vs. utility vs. privacy)

### Minimum Requirements Checklist
1. One dataset
2. At least one simple baseline (bootstrapping, noise injection, or SMOTE)
3. At least one learned generator (SDV, YData-Synthetic, or LLM-based)
4. One downstream utility task
5. A fidelity analysis **and** a privacy/memorization analysis

## Current Implementation

Dataset: UCI Adult Census Income (~32,500 rows, binary classification: income >50K).  
Downstream task: `RandomForestClassifier`, evaluated on minority class (>50K, ~24% of data) recall and F1.

Three strategies in `src/main.py`:
1. **Baseline** — RF trained on real data
2. **SMOTE** — minority class oversampled to balance, then RF
3. **CTGAN (SDV)** — fully synthetic training set via `CTGANSynthesizer`, then RF

## Known Issues & What Still Needs to Be Built

**Bugs in current code:**
- **`os.remove` on metadata.json** (`main.py:86`): crashes on fresh clone if file doesn't exist
- **One-hot encoding before SDV**: `pd.get_dummies` is applied before feeding data to SDV; SDV expects raw categorical columns, not binary dummies — this is why CTGAN fails completely (recall=0)
- **CTGAN `epochs=10`**: default is 300; at 10 epochs the GAN never learns the minority class
- **Metadata misclassification**: `workclass_State-gov` auto-detected as `administrative_unit` (PII) — should be `categorical`

**Missing for minimum requirements (items 5 from checklist):**
- Fidelity analysis: no distribution comparison, correlation analysis, or category frequency check implemented yet
- Privacy/memorization analysis: no duplicate rate, nearest-neighbor distance, or membership leakage check implemented yet

## Architecture

Everything lives in `src/main.py` as a single sequential script:

1. Load & clean `adult.data` → drop `?` rows
2. `pd.get_dummies` on features, `LabelEncoder` on target
3. 80/20 train/test split
4. Baseline RF → evaluate
5. SMOTE resample → RF → evaluate
6. SDV metadata detect → CTGAN fit → sample → RF → evaluate
7. Print minority-class recall and F1 for all three

`artifacts/metadata.json` is auto-generated by SDV and re-written every run.
